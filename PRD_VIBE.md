# 📋 Vibe Coding: 바이브 코딩 제품 요구사항 정의서 (PRD)

이 문서는 **랄프 루프(Ralph Loop)** 서브 에이전트가 성경처럼 따라야 할 마스터 가이드입니다. 에이전트는 매 루프마다 이 문서를 읽고 작업의 방향성을 결정합니다.

## 🎯 코딩 철학 (The Vibe)
- **속도보다 흐름**: 코드는 한 번에 완벽할 필요가 없습니다. 루프를 반복하며 점진적으로 완성합니다.
- **자율 치유**: 에러가 발생하면 멈추지 말고, 로그를 분석하여 즉시 수정안을 제시하고 다음 루프에서 반영합니다.
- **심미적 우선주의**: 기능뿐만 아니라 코드의 청결도와 사용자 UI의 아름다움을 최우선으로 합니다.

## 🚀 현재 목표 (Current Goal)
- [ ] 랄프 루프 환경이 정상적으로 작동하는지 확인하기 위한 헬로월드 테스트.
- [ ] 메인 페이지(`app/page.tsx`)의 디자인을 랄프로렌 스타일의 프리미엄 테마로 리뉴얼.
- [ ] 인지 재구성(`cure`) 기능의 UI를 더 부드러운 애니메이션으로 강화.

## 🛠️ 서브 에이전트 가이드라인
1. **작업 단위**: 한 번의 루프에는 하나의 의미 있는 변경만 수행합니다.
2. **커밋 메시지**: `vibe: [태스크명]` 형식으로 명확하게 기록합니다.
3. **상태 보고**: 루프 종료 전 반드시 `LIVE_LOG.md`를 업데이트하여 사용자에게 현재 상황을 보고합니다.
