#  font 적용 방식 비교 보고서: CDN vs 로컬(Local)

MoodB 프로젝트의 **전문성(의료 인터페이스)**과 **안정적인 UX**를 위해 가장 적합한 폰트 적용 방식을 분석합니다.

---

## 1. CDN 방식 (웹폰트)
Google Fonts나 외부 서버에서 제공하는 URL을 통해 폰트를 불러오는 방식입니다.

### ✅ 장점
- **설정이 매우 간편**: CSS 한 줄로 즉시 적용이 가능합니다.
- **캐싱 이점**: 사용자가 다른 사이트에서 이미 해당 폰트를 다운로드했다면, 브라우저 캐시를 통해 로딩 속도가 빨라집니다.
- **용량 절약**: 프로젝트 내부에 폰트 파일을 보관하지 않아도 됩니다.

### ❌ 단점
- **외부 의존성**: CDN 서버에 장애가 생기거나 네트워크 연결이 불안정하면 폰트가 깨져 보일 수 있습니다. (의료 인터페이스에서 신뢰도 하락 요인)
- **개인정보 보호**: 사용자 IP 주소 등의 정보가 외부 서버(Google 등)로 전송될 수 있습니다.

---

## 2. 로컬(Local) 방식 (내부 저장)
폰트 파일을 프로젝트의 `public` 폴더 등에 직접 포함시켜 서비스하는 방식입니다.

### ✅ 장점
- **최고의 안정성**: 네트워크가 오프라인이거나 불안정해도 폰트가 무조건 로드됩니다. **(의료용 앱 권장)**
- **성능 최적화**: `next/font`를 사용하면 폰트 서브셋(Subset) 최적화로 로딩 속도를 극대화할 수 있습니다.
- **신뢰도 유지**: 디자인 의도가 절대 깨지지 않습니다.

### ❌ 단점
- **프로젝트 용량 증가**: 폰트 파일 자체의 용량이 빌드 산출물에 포함됩니다.
- **초기 설정**: 폰트 데이터를 다운로드하고 설정하는 과정이 필요합니다.

---

## 3. MoodB를 위한 제안: **로컬(Next.js 최적화) 방식**

MoodB는 **"의료 + 데이터 기반 플랫폼"**이므로, 어떤 환경에서도 디자인이 깨지지 않는 **로컬 방식(Next.js `next/font/local`)**을 강력히 권장합니다.

> [!TIP]
> **Pretendard** 폰트를 로컬로 설정하면, 한글 가독성에서 타의 추종을 불허하며 의료 서비스다운 단단한 느낌을 줄 수 있습니다.
